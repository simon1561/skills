# 投研工作标准（按两档结构组织）

本文件定义普适分析纪律层。所有标准遵循**嵌入式可见**为默认形式——纪律的执行结果须自然出现在分析文本中，而不是以独立结构（清单、勾选）单独呈现。

---

## 始终生效

无论什么场景、什么分析深度，下列纪律均强制执行。

### 数据来源与推断标注

所有实质性陈述（数字类或定性类）须属于以下四类之一：

- **可查证事实**：有工具调用结果或可引用公开来源 → 须标注来源
- **合理推论**：基于可查证事实的推断 → 须标注「合理推论」并说明传导机制
- **假设**：无法验证但分析所必需 → 须标注「假设」
- **关键未知**：对判断有重要影响但当前无法获取或验证 → 须标注「关键未知」并说明对结论的影响方向（偏多 / 偏空 / 不确定）

未标注 = 默认声称为可查证事实 = 必须有可追溯来源。禁止以训练记忆替代。"大约"、"通常"、"历史上"等模糊措辞掩盖来源缺失视为违规。

**实时数据要求**：营收、利润、现金流、资产负债表、估值数据均须实时获取；行业背景、政策动态、近期新闻须通过搜索工具获取。中国市场数据分级路径：美股强制实时；A 股 / 港股优先实时，工具不覆盖时退化至最近已公告财报数据并标注截止日期；全部失败时降级为定性判断并在报告头部警示。

**口径核实**：API 返回的字段（如 yfinance 的 totalCash）不等于其字面含义。关键财务数字须标注字段名和数据来源；存疑口径（如某字段是否包含短期投资、是否含受限资金）须显式说明并交叉验证。

**一手优先、二手须判可信**：承重结论（尤其关键问题的结论）**一手优先**——以财报、电话会 transcript、原始公告等权威一手资料为准。二手信息（第三方研报、媒体报道、**以及子代理 / 委托研究的产出**）属二手来源：承重结论须回一手复核，不得把二手转述直接当一手写入结论。当某关键问题本就只有二手源（如第三方用户时长、行业份额数据），须显式做可信度判断（来源权威性 + 能否交叉印证 + 逻辑 / 常识是否自洽）并标注证据等级，不可默认其为事实。

**关键利润指标的交叉验证**：净利润、营业利润须与经营现金流（OCF）交叉验证。两者趋势明显分叉时，标注为数据红旗，并优先解释分叉原因。

**管理层口径与财报数字的对比验证**：管理层电话会、业绩说明会的表述（如"增长强劲"）须与财报数字（如 OCF 同期萎缩）对比；分叉时显式标注并优先解释。

### 数据获取与选源

获取数据前，先按「数据需求类型」选择源，而不是随手抓手边恰好可用的工具。核心原则：

- **一手优先**：监管申报 / 官方发布 > 二手聚合 API > 媒体转述（与本节「一手优先、二手须判可信」同源）
- **按需求选源**：先想清楚要哪类数据，再找对应类别的源，不被手边工具反向决定
- **撞墙升级**：目标被付费墙 / 反爬 / 动态渲染挡住时，换用浏览器类（登录态）工具，而非放弃或以记忆编造
- **关键数字交叉验证**：承接本节「口径核实」

具体的「数据需求 → 源类别 → 推荐工具 → fallback」路由表，及各工具的适用与限制，见 [data-sourcing.md](data-sourcing.md)。其中工具为**建议非强制**：环境里装了推荐工具就优先用，没有就用同类替代办法，分析能力不因缺某个工具而失效。

### 不确定性显式声明

任何对判断有重要影响但无法验证的信息，须显式声明不确定性，并说明影响方向。"我不知道"是一个合法的、有时是必须的答案。

避免用"可能"、"或许"、"在某种程度上"这类模糊措辞掩盖真实的不确定性——要么具体说"我倾向是 X 因为 Y"，要么说"我无法判断，关键依据缺失"。

**关键且不可知的变量须抬升，不能脚注化**：当推导发现结论的方向取决于某个本质上不可知的变量——不是"我没查到"，而是"再多研究也无法知道"，例如二元的监管/审批结果、创始人式公司的交接班、某条技术路线最终能否跑通——不能在挂一条「关键未知」脚注后照常给出笃定结论。须把这个变量抬升为结论的核心：结论按它的不同取值分情形陈述，让用户清楚看到"答案取决于 X——X 若如此则结论如此，若如彼则结论如彼"。「DCF 下行边界框架」是这条原则在估值/增速场景的一个特例。

### 能计算的不推给用户

凡是可以通过枚举假设和情形进行计算的问题，模型必须先计算完所有合理情形再呈现选项。**不允许把计算工作推给用户**。

典型反例（禁止）：
- "你认为合理的 CAGR 是多少？"——这是把计算推给用户
- "如果增速是 X，回报是多少？请你判断"——同上

正确做法：
- 枚举悲观 / 基准 / 乐观三档假设（参数显式给出依据）
- 计算每档情形下的具体回报（年化、绝对回报、安全边际）
- 让用户选"哪个情形最接近你的判断"，而不是"你来填参数"

### 判断级以上的结论须经校验

**结论的层级**：

- 数据级（"过去三年收入增速 30%"）→ 由「数据来源与推断标注」覆盖
- 推断级（"主营增速主要来自渠道扩张而非提价"）→ 由「不确定性显式声明」覆盖
- **判断级**（"管理层资本配置能力高于同业"）→ 须经校验
- **投资级**（"当前价格有安全边际"）→ 须经校验

**校验方法不规定**——由模型根据场景判断（可以是反例搜索、横向对比、压力测试、替代视角等）。

**但输出中须简述校验方式**——以嵌入式形式自然出现在分析中，例如："这个判断已用 X 案例反例检验，X 在条件 Y 下也确实成立"或"已与同业 A、B 的对应表现对比，本案在 Z 维度上确实更优"。不可只说"我判断 X"而不交代如何得出。

无声跳过校验等于无校验。

### 量化区间的推导透明性

任何量化区间估算（如"短期估值影响 -2% 至 -4%"、"未来 3 年 CAGR 15-25%"、"60% 概率发生 X"、"管理层执行力 7/10"）须显式给出推导依据，让读者能复核区间的方向和宽度。

**违规示例**：
- "腾讯短期估值影响约 -2% 至 -4%"——没有说明这个区间从何而来、下沿/上沿对应什么
- "未来 3 年收入增速 15-25%"——没有说明上下沿对应的假设差异

**正确做法**——至少满足以下之一：
- **公式拆解**：把估算量分解为可加可乘的因子（如："基线市场冲击 × (1 + 散户敏感度 - 承接力 - 抗跌力) = 估值冲击"，逐项给出参数依据）
- **情景对应**：明确"区间下沿对应 X 情形假设，上沿对应 Y 情形假设"
- **参数依据**：给出关键参数的来源（行业均值、历史基率、可比对象数据）

未给出推导的量化区间属于伪精确——本质等同于无来源的数字，违反「数据来源与推断标注」。区间数字越具体，对推导透明性的要求越高（"-3%"比"-3% 至 -4%"对推导的依赖更弱，因为前者就是一个点估计）。

### 报告是分析文本，不是模板填空

最终报告须有贯穿全文的核心论断；各部分服务于该论断的支撑或挑战。standards 和 scenarios 是质量检查项，不是报告章节结构。纪律名称、场景名称不得作为标题或段落框架出现在报告中。

不要把所有元素列成 bullet point 集合——综合判断章节须为连贯推理段落。

## 随场景激活

下列纪律是「判断级以上的结论须经校验」的具体方法。由框架对齐时输出的"本次激活的纪律清单"决定哪些生效，不默认全开。

每条纪律遵循嵌入式可见原则——执行结果自然出现在分析文本中，无需独立的结构化清单。

### 多层追因（适用于：解释指标异常、问题深挖）

发现任何指标异常，禁止在表层原因停止。因果链须出现在分析文本中（A 因为 B，B 因为 C……），末端须附一句判断：**再追下一层是否会改变分析结论？如果不会，可以止于此**。

深度由问题复杂度决定，不规定具体层数。简单问题可能 2 层即可，复杂问题可能 5-6 层。

参考切入维度（不限于此）：公司层（财报可见的直接原因）、行业层（同业数据验证）、政策 / 宏观 / 结构层、激励层（相关方利益诉求如何驱动该现象）。

主张因果关系时须说明传导机制（X 如何导致 Y）；只能观察相关性时，须标注「相关性，无法证明因果」。

最终给出判断：暂时性问题还是结构性 / 永久性问题。

### 横向对比（适用于：估值判断、相对优劣判断、行业分析）

解读公司指标前，先获取同业数据或行业均值。对比的具体数据和判断须自然出现在文本中——同业的具体数字、共性 vs 个性的判断——无需独立的"对比对象清单"。

须有可信的锚点（同业估值或历史基率），但不强制具体数量。标的稀缺、可比公司难以获得时，须显式声明"对比锚点稀薄"及其对结论的影响，**不强行凑数**。

### 多空对称（含 Pre-mortem 和 Pre-parade）

投资结论须同时陈述多头逻辑和空头逻辑。两侧不仅要在形式上并列，**深度也须可比**：

- **Pre-mortem**（事前验尸）：假设这个投资 3-5 年后失败了，最可能的具体失败机制是什么？须有触发条件、传导链条、终点状态。
- **Pre-parade**（事前游行）：假设这个投资 3-5 年后大成功了，最可能的具体成功路径是什么？同样须有触发条件、传导链条、终点估值锚。

如果某一侧实际就是论据稀薄（如成熟价值股的上行路径少、事件驱动股的多头逻辑单一），**显式声明该侧稀薄**，不强行凑数。

明确指出：当前证据更支持哪侧、什么事件会改变判断、最大合理下行幅度 vs 合理上行空间的赔率结构（若两者接近对称，须说明为何仍值得参与）。

### 可证伪框架 + 论断破裂唯一路径

**第一步（先回答这一条）**：什么单一情形会让整个投资论断**彻底失效**——不是降低回报，而是证明商业模式的根本判断错了？

这一条须独立陈述，与具体可追踪赌注分开——否则会被淹没在风险清单里。

**第二步**：明确 1-3 个"核心赌注"——每条赌注是一个可观察、可证伪的命题（"X 事情发生则逻辑成立，不发生则失效"）。指定具体的跟踪指标和观察时间窗口。**这些核心赌注通常正是框架对齐时识别的「关键问题」经一手深研后的固化**——前端的研究议程在此收敛为后端可证伪的命题，二者是同一条线的两端。

**第三步**：核心赌注的关键前提须按「数据来源与推断标注」的三类（可查证事实 / 合理推论 / 假设）标注——这是全文最关键的防止以推论替代事实的执行区域，须额外确认。

### 政策扫描

搜索该行业相关政策动态及已知未来政策窗口，政策对该标的的影响判断须出现在分析中。无需独立列出"已扫描政策清单"——分析里看不到政策影响的论述就等于没扫描。

同时列出已知关键未来节点（业绩发布、政策实施、管理层变动等）作为跟踪锚点。

### 反共识 + 替代视角检查

任何最终结论须做一次显式自检：

- 我的核心判断与当前市场定价隐含的假设是否一致？
- 如果不一致：差异在哪、我凭什么比市场对？
- 如果一致：这份研究的边际价值是什么？（建立跟踪基线 / 验证市场共识 / 发现市场没注意到的细节风险）

**如果第三个问题答不上来**——尝试至少一个替代分析视角（反转交易 / 成长期权 / 清算价值 / 分拆估值等），从替代视角看是否得出不同判断。允许最终回到原结论，但必须走过这个视角切换。

### 反例搜索

任何"X 意味着 Y"的因果主张建立后，主动搜索至少一个反例——历史上 X 发生但 Y 没发生的案例。

- 找到反例 → 说明"为什么这次可能不一样"或"反例条件与本案差异在哪"
- 没找到反例 → 显式声明"未发现反例（不确定是否搜索充分）"

反例搜索的执行须出现在分析文本中（"反例：A 公司在 X 条件下也曾……但本案不同在……"），无需独立标签。

### 概率推理链条

情景概率分配须附推理链条，不允许裸数字。允许两种方式之一：

- **历史基率法**：引用可比同类事件的历史发生频率为基线，再根据个案特征调整
- **显式假设法**：逐条说明"假设 X 成立则概率上调 Y%，因为 Z"

基率数据不可得时，显式声明"以下概率为纯主观假设，置信度低"。

### 不可预测变量过滤

每个分析问题在纳入报告前，先做一个简单过滤：

- 这个问题有没有可操作的答案？
- 答案会不会改变投资判断？

两个都是否的话，这个分析项不应出现在报告里。

典型例子：30 多岁创始人何时离开（不可知 + 不可预测）、宏观利率 5 年后到哪里（同上）。这些不是"key person risk"或"宏观风险"——是无效填充。

### DCF 下行边界框架（适用于：长期持有的估值分析，3 年以上）

对长期持有，**禁止用 CAGR 点估计 DCF**——这是伪精确。

须改用悲观情形下行边界框架：

- 设定明确的悲观增速假设（须显式给出依据，如"悲观 = 同业均值减一个标准差"）
- 设定明确的悲观退出估值（如"悲观 = 历史 PE 区间下限"）
- 计算悲观情形下的具体年化回报
- 判断这个下行边界是否可接受——可接受则持有，不可接受等更低价格

正面情景同样计算（基准、乐观），但**结论锚定在下行边界是否可接受**，而不是"预期 CAGR 是多少"。

### 现金底线穿透（适用于：净现金 > 市值、清算价值类主张）

任何"净现金 > 市值"或"现金底线提供保护"的主张前，必须穿透到资产负债表层级，分辨：

- 真实流动现金和现金等价物
- 短期投资和长期可变现证券
- 受限资金或非现金等价资产（运营子公司账面值、权益法投资等）
- VIE 结构下的境内 / 境外限制

对清算回收率给出判断。颗粒度由公司结构决定，不强制具体科目分类。

### 持有期信号 / 噪音过滤（适用于：长期持有，3 年以上）

框架对齐确认持有期为 3 年以上后，须显式声明本次分析中哪些信息是信号、哪些是噪音，并以此过滤后续分析的范围。

对长期持有，至少须主动排除：
- 季度数据的短期反应规则
- 分析师评级的月度变动
- 个别短期事件的买卖触发逻辑

这一过滤的结果须体现在最终结论里——不要把噪音类信息（如"Q1 数据出来后该如何反应"）写进长期持有的决策清单。

## 输出形态层

下列规范不属于纪律层（不是结论前校验），属于输出文件的结构要求。

### 多轮合并报告写作规范

合并多轮报告时，按**分析维度**组织结构，抹去"轮次"问答痕迹：

- 章节标题是描述性的分析维度（如"现金流质量"），不是针对性问题
- 结论用分析维度组织，不用问答格式
- 背景介绍用中性陈述

### 持仓逻辑框架（长期持有专属，3 年以上）

对长期持有的研究场景，在场景一完成后，**额外输出一份独立的持仓逻辑框架文件**。这份文件的功能不是研究依据，而是**持有信念的锚**——在市场大幅波动时提供快速对照，判断"论断是否还成立"。

固定结构（这是少数允许标签式可见的输出，因为快速对照需要稳定结构）：

```
1. 持仓视角：为什么持有（核心论断一句话）
2. 护城河机制：不是标签是机制——具体说明壁垒是什么、为什么难以被复制、什么条件下会被侵蚀
3. 论断破裂的唯一路径：什么单一情形让整个论断彻底失效
4. 价格框架：悲观情形下的年化回报（来自「DCF 下行边界框架」）
5. 安全边际买入价：基于悲观情形的可接受回报反推
6. 持仓纪律：什么是信号（影响持仓决策）/ 什么是噪音（不影响决策）
```

只在长期持有场景下输出。短期交易、特殊情形不需要。

### 未决问题（报告的标准组成部分）

每次研究须在报告中诚实交代本次分析的盲区。这是报告的组成部分，关于研究对象本身：

1. 本次分析仍未解决的最重要 1-3 个未知（具体说明是什么不知道、为什么难以获取）
2. 什么证据或事件能解决这些不确定性
3. 这些未知对当前结论的影响方向（若发生，偏多还是偏空）

与「不确定性显式声明」的区分：一般性的未决问题在此清单交代即可；若某个未知是**关键且不可知**的（决定结论方向），按该纪律处理——抬升为结论核心、分情形陈述，不能只塞进这个清单。
