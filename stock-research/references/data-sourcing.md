# 数据获取与选源

本文件解决一个问题：**手头有一堆数据 / 搜索 / 浏览器工具时，针对某类数据需求该优先选哪个**。它不是"必装清单"，也不是排他规定——没列到的工具只要能达到效果照样用，列到的工具没装就走 fallback。分两层：**选源原则**（稳定）和**工具优先级**（带评估日期，会随生态和手头工具变）。

---

## 选源原则

1. **一手优先,用户已给的优先级最高**：监管申报 / 官方发布 > 二手聚合 API > 媒体转述。**若用户已提供本地财报 / 文档 / 链接（重要研究中常见），优先用这些**——是用户精选、确定相关的一手原文，别舍近求远重新搜。
2. **按需求选源**：先确定要哪类数据，再找对应类别的源；别让手边恰好装了的工具反向决定你拿什么数据。
3. **撞墙升级**：目标页被付费墙 / 反爬 / 动态渲染挡住时，换登录态浏览器 / computer-use 类能力读取（按你所在客户端有什么用什么），而非放弃或凭记忆编造。
4. **关键数字交叉验证**：重要财务数字至少两源比对；二手数字回一手核（承接「数据来源与推断标注」的口径核实）。
5. **效果第一,同等效果下优先低成本**：有些工具有额度 / 收费（如 tavily、brave）。**同等效果时优先用不烧额度的工具（如自带 websearch）；但深度研究该上专业工具就上，不为省额度将就质量**——效果永远第一。

> 下表"优先用"列是**在常见已有工具中的优先序**，不是排他清单。使用本 skill 的人若缺某类能力，也可照表去补装。

## 否决记录（评估过但不建议装）

记录"评估过、不值得"的工具同样有价值——避免重复踩坑。

- **FMP（Financial Modeling Prep）**：核心用途财报 transcript 已可由"搜索 + Motley Fool 免费全文"替代；其余功能 free tier 限制重（美股 only / 仅 5 年历史 / 250 次每天 / 部分数据限头部数百只大盘股）+ 配置复杂（自托管 HTTP 服务 + 250 工具需调 Dynamic 模式）+ 与 EDGAR 大量重叠。**除非需要批量 transcript 或系统化 screener 且愿上付费版，否则不装。**

---

## 维护

工具优先级与否决记录会随生态和手头工具变化（如某 MCP 弃坑、出现更优替代、免费额度调整、客户端新增 computer-use 能力）。发现这类变化时，在「方法论反思（任务收尾时执行）」环节顺手记下，更新本表对应行与顶部"截至"日期。**源类别与选源原则一般无需改动**——它们是这套路由的稳定骨架。
