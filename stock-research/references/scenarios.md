# 场景执行规范

每个场景的输出分为**基础覆盖项**（任何情形都须包含）和**按需激活的可选项**（由框架对齐时确定的激活清单决定是否纳入）。

**重要**：基础覆盖项不是必须照顺序填写的清单。根据实际分析重点灵活调整结构和篇幅。报告是分析文本，不是模板填空。

---

## 场景一：完整公司研究

**始终激活的纪律**：S1-S5（始终生效全部）+ A2 横向对比 + A3 多空对称 + A4 可证伪框架 + A6 反共识 + A7 反例搜索

**按需激活**：A5 政策扫描（涉及监管行业时）、A8 概率推理链条（情景估值时）、A10 DCF 下行边界（长期持有时）、A11 现金底线穿透（特殊情形时）、A12 持有期过滤（长期持有时）

**取数前必须完成：商业模式第一性建立**（纯推理，不需要工具）

显式陈述这家公司靠什么赚钱（核心价值主张、收入来源结构）；护城河来源（壁垒是什么、靠什么维持、什么条件会侵蚀）；盈利质量预判（会计利润与经济利润的可能差距）；收入可持续性（可重复 vs 一次性）。

**输出基础覆盖**（顺序与详略按本次分析重点调整）：

- 投研 Summary：核心论断一句话 + 核心发现 + 估值锚 + 风险 + 验证节点
- 核心财务数据速览（实时获取，按 S1 标注口径）
- 商业模式与护城河评估
- 收入结构与驱动解析
- 利润归因（含多层追因和激励层）
- 现金流健康度（含 OCF 与净利润分叉的解释）
- 管理层质量与资本配置（5 种资本用途的历史 ROI：再投资 / 并购 / 分红 / 回购 / 还债；薪酬结构是否激励短期行为；内部人持股变动；guidance 兑现率；A 股 / 中国公司须额外检查关联交易、定增折价、分拆套现）
- 反共识视角与替代视角（A6）
- 市场定价隐含假设（当前估值隐含的增长 / 利润率假设；正常化盈利能力作为锚点；港股 / A 股双重上市的 A-H 溢价折价）
- 多空对称与赔率分析（A3，含 Pre-mortem 和 Pre-parade）
- 可证伪框架与跟踪计划（A4，先论断破裂唯一路径，再核心赌注）
- 综合判断（连贯推理段落，须有贯穿全文的核心论断）
- 方法论反思（未解决的未知 + 影响方向）

**长期持有额外输出**：A12 信号 / 噪音过滤声明 + O2 持仓逻辑框架（独立文件）

## 场景二：局部深挖

**始终激活的纪律**：S1-S5 + A1 多层追因（核心）+ A2 横向对比

**按需激活**：A5 政策扫描（指标涉及政策时）、A6 替代视角、A7 反例搜索

**输出基础覆盖**：

- 该指标 / 问题的实时数据（按 S1 标注来源和口径）
- 多层追因链（A1，因果链自然展现，末端附"再追一层是否改变结论"判断）
- 同行横向对比（A2）
- 反共识视角精简版（A6 简化版：市场对该指标的主流解释是什么、是否有被忽视的视角）
- 结构性 vs 暂时性判断
- 1-2 条证伪条件（含观察窗口）
- 关键未知（最重要的 1 个不确定性及其对当前判断的影响方向）

输出是聚焦分析段落，不强制完整报告格式，篇幅以讲清楚问题为准。

**适用规范**：O1 多轮合并报告写作规范（核心）

**输出**：一份完整 Obsidian 格式报告，结构参照场景一的覆盖内容要求。

## 场景四：持仓跟踪

**前置步骤**（必须完成再继续）：

- 询问用户："请提供上次分析报告的本地路径或附件"
- 用户提供 → 读取报告，提取"可证伪框架"章节（核心赌注 + 证伪条件 + 基线值）
- 用户无法提供 → 告知"缺少基线，建议退化为场景一（完整研究）"，经用户确认后执行

**始终激活的纪律**：S1-S5（实时数据强制）+ A1 多层追因（发现新异常时）+ A4 可证伪框架（逐条对比基线）+ A5 政策扫描（基线后新政策）

**按需激活**：A3 多空对称（投资逻辑松动时）、A6 替代视角（市场认知大变化时）

**输出基础覆盖**：

- 跟踪时间与基线来源
- 各证伪条件当前状态（✅成立 / ⚠️动摇 / ❌证伪）
- 基线后新政策、新事件、新异常
- 反共识视角变化（基线后市场主流是否变化、新信息是否改变原有非共识角度）
- 市场定价变化（基线后估值变动隐含哪些新假设、是否合理）
- 管理层动态（基线后管理层变动、大额增减持、guidance 重大调整须分析）
- 论点总体状态（仍然成立 / 需重新评估 / 建议退出）
- 下一个观察节点

## 场景五：行业 / 板块扫描

**目标**：横向对比多家公司，找出值得深研的标的，输出优先级排序而非单公司完整分析。

**始终激活的纪律**：S1-S5（多公司数据均需实时获取）+ A2 横向对比（核心）+ A5 政策扫描（行业层面强制）

**按需激活**：A3 多空对称精简版（候选标的简要多空摘要）、A6 替代视角（市场对行业主流叙事的检验）

**输出基础覆盖**：

- 扫描范围与筛选维度说明
- 横向对比矩阵（含营收增速 / 利润率 / ROE / 估值 / 现金流 / 管理层资本配置质量等关键指标）
- 行业共性 vs 公司特有判断
- 反共识视角（哪家公司的情况可能被市场误解或低估、各候选标的当前定价隐含什么假设）
- 候选标的深研优先级排序（附理由）
- 行业政策风险摘要

## 场景六：催化剂 / 事件分析

**目标**：评估具体事件对公司整体投资逻辑的影响。

**始终激活的纪律**：S1-S5 + A1 多层追因 + A2 横向对比 + A3 多空对称 + A4 可证伪框架 + A5 政策扫描（事件本身可能就是政策）+ A7 反例搜索（历史类似事件的发生频率和影响幅度）

**按需激活**：A6 替代视角（市场对事件的主流解读是否完整）、A8 概率推理链条（事件影响的不同情景）

**取数前必须完成：商业模式影响预判**（纯推理）

事件从理论上会改变这家公司商业模式的哪个环节（定价权、成本结构、护城河、客户粘性）？影响是直接的还是二阶效应？量级大还是小？先建立预判框架，再用数据验证，检验预判是否准确。

**输出基础覆盖**：

- 事件概述
- 商业模式影响预判（与实际数据对比验证）
- 各相关维度的事实 + 判断 + 量级
- 反共识视角（市场解读是否完整、是否有被忽视的中长期影响）
- 管理层响应质量（若事件涉及管理层决策）
- 同行横向影响
- 市场定价调整是否合理（事件后估值变化是过度还是不足）
- 对原有投资逻辑的净影响（正面 / 负面 / 中性 + 重大 / 一般 / 微小）
- 若有基线报告则检查是否触发证伪条件
- 建议动作

## 场景七：特殊情形分析

**典型触发**：净现金 > 市值、清算价值大于市值、潜在私有化、Sum-of-Parts 大于市值、烟蒂股

**始终激活的纪律**：S1-S5 + A2 横向对比 + A3 多空对称 + A4 可证伪框架 + A6 替代视角 + A7 反例搜索 + A11 现金底线穿透（核心）

**按需激活**：A5 政策扫描（涉及私有化退市等监管事件时）、A8 概率推理链条（情景概率分配时）

**取数前必须完成：特殊情形假设的完整性检验**

特殊情形分析依赖一个核心假设（如"现金可被股东获取"、"清算价值能被回收"、"私有化会发生"）。先把这个假设显式化，再分析它在什么条件下成立 / 不成立。

**输出基础覆盖**：

- 特殊情形论点的核心假设（显式陈述）
- 现金 / 资产穿透分析（A11，分辨真实流动 vs 受限 / 非现金等价）
- 假设成立的条件分析（VIE 限制、监管约束、管理层意愿、催化剂时间窗口）
- 反例搜索（A7：历史上同类型情形最终的归宿分布）
- 同业横向对比（A2：可比情形的当前估值或私有化溢价）
- 多空对称（A3：上行催化剂 vs 下行陷阱）
- 概率推理链条（A8：不同情景的概率分配及推理依据）
- 替代视角（A6：从市场为何"错误"的角度反推论点是否成立）
- 投资级结论（持仓建议 + 仓位建议 + 退出条件）

特别注意：特殊情形分析最容易出现"标签式正确，实质错误"的陷阱（如把 totalCash 当作纯现金）。S1 口径核实 + A11 现金穿透 + A7 反例搜索三者必须实质执行，不可仪式化。

## 提交前自查（所有场景适用）

报告提交前，模型在内部确认（不要求把核查答案写进报告——核查的对象本身应已嵌在分析里）：

- 所有事实性陈述是否有来源标注？所有推断是否标注了类型？（S1）
- 不确定性是否显式声明？（S2）
- 该计算的是否都计算了？没有把判断推给用户？（S3）
- 判断级以上的结论是否在文本中体现了校验过程？（S4）
- 报告是否有贯穿全文的核心论断？（S5）
- 本次激活的随场景纪律是否都在分析文本中有体现？

任何一项答否，回到分析阶段补全，再提交。
