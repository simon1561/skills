# stock-research skill

> 一个将投研工作的元方法论编码为模型约束的 Claude Code skill

---

## 设计哲学

**本质**：Skill 是将投研工作的元方法论（严谨性、可校验性、置信度自管理）编码为模型约束的工具。具体投资风格不预设，由每次框架对齐捕获——同一个 skill 能处理价值投资、特殊情形、行业扫描等不同场景。

**AI 的角色——好下属模型**：AI 是完整的分析执行者，完成包含信息搜集、推理、判断、结论的完整分析，自主管理置信度（高置信直接给结论 / 有倾向但不确定标注待确认 / 需要用户输入才能推进时明确提问）。不做信息搬运员，也不把所有判断甩给用户。

**约束的两个位置**：
- 普适分析纪律层（始终生效纪律 + 随场景激活纪律）——结论之前必须走过的校验步骤
- Session 方法论贯穿层——框架对齐捕获的目标、方法论、持仓背景，主动校准后续所有分析

**规则必须产出可见输出**：默认采用"嵌入式可见"（纪律的执行结果自然出现在分析文本中）。能被无声跳过的规则等价于不存在。

## 适用边界

**强普适场景**（结构和纪律直接适用）：
- 完整公司研究（任何投资风格）
- 单一具体问题（如"为什么毛利率下降"）
- 行业分析
- 特殊情形分析（如净现金 > 市值的烟蒂股）
- 持仓决策（加仓 / 减仓 / 卖出判断）

**部分适用场景**（结构适用但纪律执行方式需调整）：
- 宏观 / 政策分析（横向对比对象不同）
- 多标的对比筛选（校验对象是相对优劣）

**不适用场景**：
- 纯数据查询（"苹果当前 PE 是多少"——没有判断级结论）
- 研究方法论本身的元问题
- 组合层面的资产配置

## 能力边界

### 下限（可以保证做到的）

对任何一家公司，从零开始，产出一份不犯低级错误、不遗漏结构性问题的分析：财务数据交叉验证、传导机制识别、多空对称、关键未知显式标注、判断级结论经校验。

### 上限（最好情况下能做到的）

在用户完全不了解某家公司时，帮助在短时间内建立一个有质量保证的认知起点，并识别出领域内分析师也可能忽略的结构性机制问题。

### 一句话定位

skill 是分析起点的质量保障工具。它能帮助站对起跑线、把问题想清楚、给出有依据的结论；但最终的投资 conviction 仍来自用户的行业积累和持仓判断——好下属交出完整报告，拍板的仍是上级。

### v0.7.0 — 整体性重组：恒定底座 + 研究生命周期（2026-06-06）

**背景**：v0.6.x 多轮小修补累积成"地质层"——新规则总塞进现有结构最近的位置，未整体重组。本次做架构层面的整体性重构。

**诊断**：skill 有两个正交维度（恒定轴=什么是好分析 / 时序轴=一次研究怎么走），但线性九节把它们交错了，导致：防漂移散落三处+重复、元活动（方法论反思/未决问题）错位塞进 standards、纪律双维护漂移（SKILL 摘要 6 条 vs standards 7 条不一致）、概念重复（置信度/输出原则）。

**改动（采用「折中」方案）**：
- **SKILL.md 重组为两部分**：恒定底座（定位 / AI 角色与置信度 / 纪律体系总入口 / 场景路由）+ 研究生命周期（对齐期 → 执行期·贯穿校准与防漂移 → 收尾·方法论反思）。九节 → 七节。
- **防漂移统一**：原散落的 Session 贯穿 / 检查点 / 提交自查整合挂到生命周期节点上，回应设计文档"缺时间性维度"的开放问题；检查点一/二/三 → 语义时点名（对齐重述 / 判断前重读 / 交付前自查）。
- **standards 提纯**：未决问题 → 输出形态层；方法论反思 → SKILL 收尾节（standards 不再重复）。
- **消除双维护 + 概念重复**：SKILL 纪律段只列名 + 指针（描述唯一在 standards，修复 6vs7 不一致）；置信度（角色 + 表达）合并；输出原则去重；修复 v0.6.7 去编号的一处残留。

不改任何纪律的**内容**，只重组结构。**适中 / 折中 / 彻底三方案的完整权衡 + 彻底方案形态（供未来升级）见设计文档 `Skill设计哲学和迭代思路.md` 第六节。**

### v0.6.6 — 关键问题前置 + 一手优先二手判可信（2026-06-01）

**背景**：微博持股收息研究 session 暴露两处系统性问题——(1) 用户基础是该 thesis 的承重判断，但模型第一版未对其做一手纵深（依赖竞品二手数据、未动用已下载的年报），需用户主动提示才补深；(2) 把子代理的二手转述（"投资组合 67% 不透明、$11 亿非上市股权"）直接当事实写入报告，事后经一手年报核实证伪（真相为权益法 $12.1 亿含上市标的 + 成本法非上市仅 $2.9 亿）。两者共因：框架对齐没有显式锁定"哪些是决定结论的关键问题"，研究执行也没把"一手优先、二手须判可信"约束到位。

**改动**：
- **SKILL.md 框架对齐第一轮**：新增「识别并与用户拉齐关键问题」步骤——基于现状校准 + 目标 + 商业模式第一性，识别 3–6 个决定结论方向的关键问题（定性定量皆可），拉齐后作为深研议程；明确此步只识别问题、不要求列一手证据（深研阶段才产出）。检查点一加入"重述关键问题"。
- **standards.md S1**：新增"一手优先、二手须判可信"——承重结论一手优先（财报 / 电话会 / 原始公告）；二手信息（含子代理 / 委托研究产出）属二手来源，须回一手复核，或在只有二手源时显式判可信度并标证据等级。
- **standards.md A4**：补承接句——核心赌注通常由框架对齐的"关键问题"经一手深研后固化，前端议程与后端可证伪命题串成一条线。

属补全既有方法论节点（框架对齐的关键问题锁定 + S1 的二手来源约束 + A4 串联），非新增独立规则层。

### v0.6.4 — 新增 S5 量化区间推导透明性（2026-05-24）

**背景**：跨境券商八部委整治事件分析 session 暴露——给出"-2% 至 -4%"这类量化区间估算却未拆解推导逻辑，读者无法复核区间宽度和方向，等同于伪精确。原 S1（数据来源标注）覆盖了"事实陈述须有来源"，但量化估算的"区间须有推导"未被显式约束。S3（能计算的不推给用户）说"枚举所有合理情形再算结果"，但没要求"展示推导链路"。

**改动**：

- 原 S5（报告是分析文本）顺延为 S6。
- scenarios.md 同步更新所有 S1-S5 引用为 S1-S6；提交前自查清单加一条"所有量化区间是否给出推导依据？（S5）"。

### v0.6.3 — 框架对齐的持有期处理补全（2026-05-22）

**背景**：英伟达研究 session 暴露一处系统性执行漂移——多标的项目把"统一按 3 年持有"设为研究基准（为让标的同尺度可比），模型却把它当成真实估值期限，用"三年内兑现不了"为由折掉了标的的长跑道价值。框架原先只规定"长期 → 过滤短期噪音"，没规定"基准期限 ≠ 价值上限"，节点不完整。

**改动**（SKILL.md 框架对齐，两处）：
- **捕获环节**：捕获持有期时须分清它是"真实估值期限"还是"跨标的可比的研究基准"。
- **Session 方法论贯穿**：新增一条——持有期是基准、不是价值的上限；不得因某项价值 / 期权"超出名义持有期"就折掉它；可能是长久期复利的标的须另叠加无期限持有视角。与既有"长期 → 过滤短期噪音"互补成对。

属补全既有方法论节点（持有期如何校准下游分析），非新增独立规则层。

### v0.6.2 — 方法论反思重构 + 关键不可知变量原则（2026-05-21）

**改动一·方法论反思拆分**：原"研究结束后的方法论反思"名实不符——它实际输出的是"对研究对象的未决问题"，而非对 skill 本身的反思。拆为两部分：
- **未决问题**：关于研究对象的盲区，留在报告里（改名，不再叫"反思"）
- **方法论反思**：关于 skill 本身的复盘，任务收尾时由模型主动执行——结构化复盘本次 session 用户的每一处纠正/反驳/不满，逐条判定是方法论失误（skill 可优化）还是执行失误，输出到对话由用户裁决

**改动二·关键且不可知变量原则**（S2 扩充）：当推导发现结论方向取决于某个本质不可知的变量（如二元监管结果、创始人交接班），不能挂个「关键未知」脚注后照常给笃定结论，须把该变量抬升为结论核心、分情形陈述。A10（DCF 下行边界）是这条原则的一个特例。

### v0.6.1 — 框架对齐增加现状校准（2026-05-21）

**背景**：框架对齐阶段，模型基于内部知识推理出研究框架供用户确认。但任何模型的内部知识对股票的时效性信息都是过时的——股价、最新财季、近期事件随时在变。基于过时认知推理的框架不可靠，用户确认它反而制造"已对齐"的假象。

**改动**：在框架对齐前增加强制的"现状校准"前置步骤——调用股票数据和搜索工具，获取当前股价/市值、最新财季、近期重大事件、结构变化，使提交给用户的框架建立在最新事实上。现状校准有明确边界（不做完整数据获取和深度分析），完整分析仍须等用户开始指令。第一轮对齐输出新增"现状快照"作为可见检查点。

## 待解决的设计问题

- **方法论层面的开放问题**：设计哲学目前是"空间性"的（约束在哪一层），缺少"时间性"维度——长链分析中的持续生效、多轮对话的对齐保持、中途反对的响应机制、模型置信度自校准的可靠性。详见设计哲学源文档第三节。

- **NotebookLM 集成**：工具尚未安装部署。集成方向：作为公司内部文档（年报/中报/电话会纪要）的全文检索层，补充 yfinance 无法覆盖的子科目明细和附注信息。

- **规则最小化**：随使用积累，持续识别哪些规则是真实失败模式的记录、哪些是防御性过度工程，后者逐步删除。
