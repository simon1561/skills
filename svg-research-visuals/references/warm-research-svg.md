# 精致暖色 SVG 研报图规范

创建或修改研究型SVG时都应阅读本文件。

## 设计理念

专业的暖色感来自**低饱和与协调**，而不是把色调做冷。应使用克制的暖色、清楚的信息层级和足够的内容密度，使图像像一张经过设计的研究页。

每张图都必须承担一个判断：

- 不好：装饰性图标或流程图，只是重复了旁边的文字。
- 好：单页图自身就能承载论点、证据、对比和结论。

## 默认页面原型

较完整的图默认采用 `1600 × 1000`。

推荐结构：

1. 克制但清楚的大标题：核心判断。
2. 副标题：机制、范围或限制。
3. 细分隔线。
4. 主体内容，可选择：
   - 两个主要面板；
   - 三张并列卡片；
   - 左侧梯度结构、右侧证据面板；
   - “成立条件 / 证伪信号”对照。
5. 底部深炭色结论栏：投资含义或胜负手。

边距约 `70px`，面板间距约 `45–60px`，卡片圆角约 `10–14px`。

## 色彩令牌

| 角色 | 颜色 | 用途 |
|---|---|---|
| 背景 | `#f8f5ef` | 暖米白画布，不使用纯白 |
| 卡片底 | `#fdfaf5` / `#ffffff` | 信息区块 |
| 锚色 | `#2e2a25` 配 `#b08d5a` 描边 | 核心论点、底部结论或唯一中心锚点 |
| 锚点高亮 | `#f0b860` | 深色锚点中的关键短语 |
| 锚点文字 | `#f0e8da` / `#d9bf9a` | 深色锚点中的正文与次级文字 |
| 单色赭石序列 | `#7a4a28`、`#94612f`、`#ab7642`、`#c08c52`、`#d8ad7a`、`#ecdcc6` | 梯度、质量层级、强弱序列 |
| 分类色 | `#c17a45`、`#9a8a55`、`#6f7d67`、`#6c7d89`、`#8a6f78` | 并列分类；灰蓝仅作低饱和对比锚 |
| 参照线 | `#c9bca9`、`#d7ccbc` | 多系列图中的背景同行、辅助序列 |
| 警示色 | `#9a5048` | 风险、反转或警示，小面积使用 |
| 警示底色/文字 | `#f4eae7` / `#8a443c` | 提示框 |
| 标题文字 | `#2f2417` | 主标题 |
| 正文文字 | `#3d3122`、`#5a4d39`、`#7d6e54` | 正文、次级文字、注释 |
| 分隔线 | `#e3d4bd` | 细分隔线 |
| 标签底 | `#f3e9da`，文字 `#9c5f2a` | 小标签 |

规则：

- 连续强弱或质量层级使用同一赭石色相的明度梯度。
- 并列分类使用多个低饱和分类色。低饱和灰蓝可以作为并列对比锚；专业感来自低饱和与协调，不要求所有分类都局限在暖色相。
- 多系列图若有明确研究主角，主角使用锚色或赭橙，同行优先压成浅赭灰参照线；不要为了“用满色板”让所有系列争抢注意力。
- `#9a5048` 是有明确风险语义的警示色，不得作为普通分类色或为了凑足系列数量而使用。
- 小倍图中，对象名称已承担识别功能时，默认所有面板使用同一中性赭石色；只有存在明确研究主角时，才将主角改用赭橙，其余面板保持中性色。
- 警示框使用浅警示底色加窄红色左边条；不要把整张面板染红。
- 深炭色用于锚定，不用于占据页面。优先只使用一个底部结论栏或一个核心论点区。

## 字体与字重

字体栈：

```svg
"PingFang SC","Microsoft YaHei","Noto Sans CJK SC",sans-serif
```

对于 `1600 × 1000` 的PPT式SVG：

| 层级 | 建议字号 | 字重 | 说明 |
|---|---:|---:|---|
| 主标题 | `48–52px` | `800` | 清楚但不过分像海报 |
| 副标题 | `23–26px` | `400–500` | 解释性、退后；中文偏重时用 `400` |
| 分区标题 | `29–32px` | `800` | 暖色强调 |
| 卡片标题 | `22–25px` | `700` | 不要让所有卡片都显得很重 |
| 正文 | `19–21px` | `400` | 默认常规字重；公式可单独用 `700` |
| 注释 | `15–18px` | `400` | 应明显退后 |
| 标签文字 | `17–18px` | `600` | 紧凑分类标签，不是第二级标题 |
| 底部结论高亮 | `24–27px` | `800` | 只强调最关键的一句 |
| 底部结论正文 | `20–22px` | `400–500` | 只让关键短语加粗 |

对于 `1040 × 720/800` 的小图，整体缩小约20–25%。

PingFang SC 的最高现成字面通常是 Semibold；在当前 macOS 字体匹配中，`600/700/800` 可能都落到同一字面。这些数值仍可保留为语义层级，但不要指望 `700` 与 `800` 稳定地产生肉眼可见的差异。主标题、分区标题和关键数字应同时依靠字号、颜色、位置和留白建立层级。

避免：

- 正文到处使用 `600/700` 粗体。
- 在渲染偏重时，把中文小字默认设为 `500`。
- 标题过大而吞噬页面。
- 负字距。
- 没有换行的密集段落。

## 垂直节奏与基线网格

应在放置具体文字前设计好垂直节奏。精致页面的规律感来自共享网格，而不是凭眼睛逐个调整间距。

对于 `1600 × 1000/1100` 的研究页：

- 尽可能使用 `8px` 间距体系，常用间距为 `16 / 24 / 32 / 40 / 48px`。
- `19–21px` 正文采用 `34–38px` 基线间距，卡片证据行默认 `36px` 很稳妥。
- `15–17px` 脚注与上一行正文基线之间约留 `24–28px`。脚注可以比完整正文行更近，但所有同类卡片必须采用相同间隔。
- 并列卡片要先定义共享的基线行。对应元素在不同卡片中必须使用相同的 `y` 值。
- 段落间距与行距分开处理：新证据块可以有更大的有意间隔，同一证据块内的行距必须规律。
- 某张卡文字更多时，先缩写或重组；不要靠逐步挤压行距解决。
- 普通提示框左右内边距至少 `24px`，正文的实际文字边界距描边至少 `20px`；内容过长时扩宽、换行或缩写，不要贴边容纳。

三张并列卡片可使用如下网格：

```text
标签       y = 324
标题       y = 378
关键数字   y = 432
分隔线     y = 455
第1行      y = 493
第2行      y = 529
第3行      y = 565
第4行      y = 601
脚注       y = 628
```

具体数字可以变化；关键是所有卡片共享同一组行，并使用有限的间距词汇。

## 绘制顺序与局部对比度

SVG按文档顺序绘制。一个元素即使设计正确，若在文字或数据之后绘制，也可能破坏视觉效果。建议的层级顺序：

1. 画布和卡片背景；
2. 区域高亮、网格线和低对比度辅助线；
3. 数据面积、条形、流程路径和结构连接线；
4. 核心折线、数据点、箭头和关键数值；
5. 标签、注释和提示框。

实用规则：

- 面积填充先于折线和数据点绘制。折线是主序列，不能被自己的填充层削弱。
- 表格高亮列或图表色带要先于其中的文字和数字绘制，不要事后用半透明矩形压在文字上。
- 箭头头部必须与路径使用相同的语义颜色。红色风险分支配金色箭头头部会制造混乱信号。
- 离散时间线的日期顺序本身已表达方向；当节点使用圆点时，不要再让箭头头部压在终点圆点上。连线应置于圆点下方并以圆点作为终点锚；箭头留给因果、资金或决策流程。
- 检查局部对比度，而不只是全局配色。标签在整张图里可读，不代表在染色带或深色卡片上仍可读。
- 强调时谨慎使用透明度。若需要较深的覆盖层才能突出内容，应改用结构、描边或字重，而不是继续叠加深色遮罩。

## 常用版式模式

### 研究结论页

适合核心投资逻辑：

- 顶部标题与副标题；
- 两个主要面板或三张卡片；
- 面板内部的证据子块；
- 底部结论栏写明胜负手、倾向判断或证伪信号。

### 左侧梯度、右侧证据

适合质量层级、成熟曲线、货币化阶梯：

- 左侧：单色明度梯度；
- 右侧：三张证据卡或验证条件；
- 底部：区分领先指标与真实价值的结论栏。

### 条件与证伪

适合投资判断框架：

- 顶部深色锚点：核心论点；
- 左侧：成立时应看到什么；
- 右侧：出现什么要降低置信度；
- 底部：一个明确的临界结论。

### 资产桶 / 并列分类

适合资本开支、业务层、利益相关方：

- 三张并列卡片配分类色边条；
- 可比字段放在相同的垂直位置；
- 底部论点栏把分类重新连接为一个投资含义。

### 结论栏与来源槽位

- 结论栏只用于需要提炼判断的研究图；纯功能图、数据查阅图和中性流程图不强制添加。
- 结论栏默认直接写一句核心判断，不额外添加“一句话结论”等重复标签。确有必要时可增加一行较轻的解释，但不能把结论栏写成正文段落。
- 来源行固定放在画布底部，字号 `15–17px`、常规字重、左对齐。推荐格式：`来源：公司财报、数据平台｜截至 YYYY-MM｜E 为估算`；虚构样图写明 `数据为虚构`。

## SVG 样式模板

以下模板可作为起点，再根据实际版式调整尺寸：

```svg
<style>

  .sub { font-family:"PingFang SC","Microsoft YaHei","Noto Sans CJK SC",sans-serif; font-size:24px; font-weight:400; fill:#6b5c45; }
  .h { font-family:"PingFang SC","Microsoft YaHei","Noto Sans CJK SC",sans-serif; font-size:30px; font-weight:800; fill:#9c5f2a; }
  .h2 { font-family:"PingFang SC","Microsoft YaHei","Noto Sans CJK SC",sans-serif; font-size:23px; font-weight:700; fill:#2f2417; }
  .body { font-family:"PingFang SC","Microsoft YaHei","Noto Sans CJK SC",sans-serif; font-size:20px; font-weight:400; fill:#3d3122; }
  .note { font-family:"PingFang SC","Microsoft YaHei","Noto Sans CJK SC",sans-serif; font-size:17px; font-weight:400; fill:#7d6e54; }
  .pill-label { font-family:"PingFang SC","Microsoft YaHei","Noto Sans CJK SC",sans-serif; font-size:17px; font-weight:600; fill:#9c5f2a; }
  .formula { font-family:"PingFang SC","Microsoft YaHei","Noto Sans CJK SC",sans-serif; font-size:21px; font-weight:700; fill:#7a4a28; }
  .num { font-variant-numeric:tabular-nums lining-nums; font-feature-settings:"tnum" 1,"lnum" 1; }
  .card { fill:#fdfaf5; stroke:#c17a45; stroke-width:3; }
  .anchor { fill:#2e2a25; stroke:#b08d5a; stroke-width:2.5; }
  .pill { fill:#f3e9da; }
  .warn { fill:#9a5048; }
</style>
```

提示框模式：

```svg
<rect fill="#f4eae7" stroke="#e3cdc7" stroke-width="1.5"/>
<rect fill="#9a5048" width="8"/>
```

## 视觉质检清单

使用Chrome渲染：

```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless --no-sandbox --disable-gpu \
  --force-device-scale-factor=1 \
  --hide-scrollbars \
  --screenshot="/tmp/out.png" \
  --window-size=<width>,<height> \
  "<file.svg>"
```

检查PNG。若出现任一问题，交付前必须修正：

- 标题过于吵闹或像海报。
- 正文和注释看起来全部加粗，常规文字无法明显退后。
- 并列卡片没有逐行共享基线。
- 行距混用多组无关数值，而不是有限间距体系。
- 面积填充、半透明高亮或彩色带位于折线、数字或标签上方并削弱它们。
- 箭头头部和连接线使用相互冲突的语义颜色。
- 网格或数据侵入标题、单位或注释区。
- 层级图同级节点在完整文字渲染后接触或重叠。
- 连续时间线对时间间隔明显不同的事件采用相同间距。
- 小倍图中的可比面板使用不同Y轴范围，导致斜率不可比。
- 构成迁移图要求读者从波动基线中精确读取关键中间层，却没有抽出独立占比折线。
- 比率变化混用 `pct` 与 `ppt`，或并列数字没有启用等宽数字。
- 预测期仅靠年份后的 `E` 区分，没有虚线、预测带或直接标注。
- 使用警示赭红作为普通分类色，导致无依据的风险暗示。
- 文字贴近卡片边缘或在底部被裁切。
- 提示框正文距描边不足 `20px`，或只因“技术上未溢出”就接受零内边距。
- 卡片过空，或段落过于密集。
- 把原文段落逐一装进卡片，却没有识别事实、机制、证据、限制和结论。
- 用箭头表示单纯阅读顺序，或在压缩长文时丢失替代解释与不确定性。
- 论点树只有正面支撑，没有与各分支对应的开放风险或证伪条件。
- 把普通线性流程排成一圈，却没有写清返回起点的反馈机制；或让飞轮连线穿过标签和节点。
- 用熟悉但错误的简称包装派生指标，例如把“价格 ÷ 税后经营利润”写成P/E，或把目标ROIC门槛写成会计盈亏平衡线。
- 中文金额混用美元符号与“亿/万亿”等中文数量单位，导致单位体系不统一。
- 深炭色覆盖页面面积过大。
- 连续数据误用分类色，或分类数据误用让人困惑的渐变。
- 警示红过饱和或覆盖面积过大。
- 图缺少底线判断；若图是纯功能数据图，则应清楚回答其预设问题。

不要使用 `qlmanage -t`；它可能裁切宽幅SVG。
