# 信息关系到版式：SVG 版式选择手册

这份手册解决的不是“怎么画得好看”，而是“什么信息应该长成什么样子”。视觉形式必须服从信息关系；选错形式，再漂亮也会增加读者的理解成本。

## 先选阅读场景

| 场景 | 建议画布 | 设计重点 |
|---|---:|---|
| Obsidian / 横向文档 | 1600 × 1000 或 1600 × 1100 | 信息密度适中，正文不小于 18px |
| 16:9 演示页 | 1600 × 900 | 留白更多，单页只传达一个结论 |
| 聊天转发 / 手机阅读 | 1080 × 1350 | 纵向阅读，最多 2–3 个模块，减少脚注 |
| 文内小图 | 1040 × 720/800 | 去掉次要说明，保留标题、核心关系和必要注释 |

不要把横向长图直接缩小给手机看；也不要为了塞进固定画布而把正文缩到读不清。

## 信息关系与推荐形式

| 读者需要看懂什么 | 首选形式 | 何时不用 |
|---|---|---|
| 单一结论及其支撑 | 结论页：主数字 + 2–3 条证据 | 没有足够证据时，不要做成“结论海报” |
| 两组或三组对象的同字段比较 | 对比矩阵 / 平行卡片 | 比较对象超过 5 个时，改用表格或条形图 |
| 谁大谁小、差距多大 | 排序条形图 | 类别无自然排序且差异很小时，用表格 |
| 一个整体由什么构成 | 堆叠条形图 / 100% 堆叠条形图 | 类别超过 5 个或数值接近时，不用饼图 |
| 随时间如何变化 | 折线图 / 时间线 | 时间点稀少且没有连续趋势时，改用事件时间线 |
| 多家公司同一指标如何变化 | 多系列折线 / 主角＋参照 | 系列超过六条或需要同等重视时，改用小倍图 |
| 多家公司同一指标但无法在一张图清楚容纳 | 小倍图 | 指标或尺度不同且未归一化时，不能强行共享坐标轴 |
| 一个整体的构成如何随时间迁移 | 100%堆叠柱形图 / 面积图＋关键层折线 | 只关心一个关键层时，直接使用占比折线 |
| 因果、资金、产品或决策如何传导 | 有向流程图 | 没有方向关系时，不要画箭头 |
| 层级、归属、组织或产品树 | 树图 / 缩进层级图 | 节点过多时，拆成两层，不要画密集蜘蛛网 |
| 多空、条件、取舍或二元张力 | 左右对照 / 条件与证伪图 | 双方不是对立关系时，不要强行对半分 |
| 估值或利润从起点到终点如何变化 | 瀑布图 / 价值桥 | 只有静态构成时，使用堆叠条或分部加总 |
| 两个估值假设如何共同影响结果 | 敏感性矩阵 / 热力格 | 只有单变量变化时，使用情景条形图或折线图 |
| 两个变量共同决定位置 | 2 × 2 矩阵 / 散点图 | 坐标含义模糊时，不要用 2 × 2 制造伪精确 |
| 多指标的当前状态 | KPI 仪表板 / 小倍图 | 指标没有可比基准时，先说明基准，不要堆数字 |

### 选择原则

- **时间用横轴，数量用长度，方向用箭头，层级用缩进。** 不要为了视觉变化而替换这些读者已经熟悉的编码。
- **卡片只适合并列的离散信息。** 卡片不适合表达连续趋势、精确排名或复杂流向。
- **一张图只保留一个主阅读路径。** 可以有补充信息，但读者应在三秒内知道从哪里开始、最后得到什么。

## 各类图的最低规范

### 数据图

- 明确单位、时间范围、数据来源和是否为估算。它们应贴近图表，而不是藏在文档外。
- 条形图默认从零开始。若必须截断，画出清晰的轴断裂或在标题中注明，不能靠视觉误导放大差距。
- 类别默认按数值、时间或业务逻辑排序；不要按录入顺序排。
- 直接标注关键数据点优先于远离图表的图例。图例不是装饰，读者不应在画布两端来回找颜色含义。
- 不用饼图展示超过五类、差异很小或需要精确比较的组成。
- 避免双Y轴；若无法避免，用颜色和单位强区分，并说明两个轴并非可直接比较。
- 把标题、单位和方法注释放在独立的图表标题区；最高网格线、数据标注和绘图区不得侵入该区域。图表“画得进去”不代表读者能分清图例与数据。
- 对连续时间轴，横向距离必须与真实时间间隔成比例。若事件之间的间隔不同却采用等距布局，应明确这是“类别顺序”而不是时间尺度，避免用视觉距离制造错误节奏感。
- 对有离散日期圆点的时间线，圆点是节点和终点锚，时间顺序已说明方向；不要在圆点上叠加箭头头部。只有需要额外表达因果或资金传导时，才使用箭头。

### 多系列时间比较

- `2–4` 条同等重要的系列可以使用分类色，但仍应通过线宽、点形或直接标注提供第二重识别线索。
- `5–6` 条系列优先采用“主角＋参照”：研究标的使用 `4–5px` 主色实线；关键对手可使用 `3px` 低饱和灰蓝；其余同行使用 `1.5–2px` 浅赭灰线。若所有系列都同等重要，改用小倍图。
- 超过 `6` 条时，不要继续堆叠折线；改用小倍图、相对指数图、热力表或排序表。
- 优先在末端直接标注系列名称与最新值，并在绘图区右侧预留标签轨道。标签相距不足一行时，使用短引导线和固定错位，不要退回远端图例让读者来回查色。
- 同业比较必须保证单位、期间和会计口径一致。颜色丰富不能掩盖口径不可比。

### 小倍图

- 同一指标的同行小倍图必须共享时间范围、Y轴最小值、最大值和网格位置；每个面板各自缩放会制造不可比的斜率，是比配色错误更隐蔽的误导。
- 折线小倍图允许截断Y轴，但所有面板必须采用完全相同的截断范围，并在图中明确标注。若数值跨度过大，改用基期指数或分组展示。
- 面板按最新值、期间变化幅度或明确的研究逻辑排序，不能沿用录入顺序；排序依据应写在副标题或注释中。
- 每个面板直接标注对象名称与最新值，不使用远离面板的统一图例。只在第一列显示Y轴刻度是可接受的，但所有网格必须逐行对齐。
- 单页以 `4–9` 个面板为宜；超过 `9` 个时改用热力表、排序表或拆页。不同单位的指标不能共享尺度，应先归一化或拆成不同组。

### 构成随时间变化

- 年度、季度等离散时间优先使用100%堆叠柱形图；时间点较多、强调连续迁移时可使用100%堆叠面积图。
- 层数不超过 `5`，层序在全期间保持固定；规模最大、最稳定或最具业务基底意义的大类优先贴底。
- 在末端直接标注各层名称和最新占比。构成图只表达100%内部迁移；若还要表达总量增长，应另加总量折线或另画一张图。
- 中间层的基线会随下方各层变化，读者无法精确读取其自身趋势。若某一中间层承载核心论点，必须把它抽出为独立占比折线；堆叠图负责全景，独立折线负责精确变化。

### 流程图与机制图

- 每条箭头都应回答“什么从哪里流向哪里、为什么”。箭头没有语义就删掉。
- 方向固定：默认从左到右或从上到下，整张图不要混用。
- 节点数量超过七个时，先分组或分层。读者应该先看到主链路，再看到例外和支线。
- 例外、风险和反馈回路必须与主流程视觉区分，但不能用大面积警示色淹没主线。

### 层级图

- 先按最宽的一层计算节点宽度、间距和画布边界，再画父节点与连线。不要先排上层、最后把叶子硬塞进去。
- 同级节点之间保留明确可见的间距；对1600px画布，常规叶子节点至少留 `24–32 px` 空隙。标签较长时先缩短文字或扩大画布，不要允许盒子相叠。
- 连线先绘制、节点后绘制。连接关系应被看见，但不能穿过卡片、压住文字或成为最醒目的元素。

### 估值、比较与情景页

- 所有并列模块使用相同字段顺序与相同基线；否则读者无法横向比较。
- 基准情景必须视觉最稳定；上行和下行只用小面积颜色区别，避免把倾向性包装成事实。
- 估算值、会计值和市场价格必须写清口径，不能用同一种视觉权重混在一起。
- 关键假设、敏感变量和结论分开排。数字不是结论，读者需要看到数字变化如何传导到判断。

#### 敏感性矩阵

- 默认使用 `5×5` 或 `7×7`，横纵轴直接写明假设、单位和变化方向；格子过多时优先缩减情景，不要压小数字。
- 基准情景锚在实际假设交点；只有矩阵围绕基准对称构造时，它才位于几何中心。基准格使用深炭描边与暖金内框，不能只靠填色识别。
- 低于基准值使用低饱和赭红梯度，高于基准值使用赭石梯度，基准附近保持米白中性。颜色表示结果高低，不自动等同于风险等级。
- 每格直接写数值，色标紧贴矩阵。估值金额、每股价值和百分比必须明确单位。

### 表格与信息卡

- 表格适合精确查阅，卡片适合提炼判断。需要同时比较多行多列时，优先表格。
- 列标题、数值和单位应对齐。文本左对齐，数值右对齐；启用 `font-variant-numeric: tabular-nums`，并使用统一 `text-anchor="end"`。若小数位数不同且必须严格按小数点对齐，应拆分整数和小数部分或统一显示精度。
- 信息卡每张只承担一个角色：结论、指标、证据或注释。混合角色会削弱层级。
- 同行卡片的顶部、主标题、关键数字、正文行和脚注使用共享基线网格。
- 比率本身使用 `%`；百分点变化使用 `ppt`，中文正文可写“个百分点”。不要在同一套图中混用 `pct`、`ppt` 表示百分点变化。

## 密度、留白与文字

- 先保留边距和行距，再削减文字；不要先压缩字号。若内容装不下，拆页、缩短文案或换图形结构。
- 同一张图只使用一套正文行距。正文、脚注和标题可以有不同节奏，但同一层级不得任意变化。
- 一张1600px宽的研报页通常最多容纳三张高信息密度卡片；更多模块需要更简单的内容或改成表格。
- 中文小字的常规字重为400。标题和关键数字承担视觉锚点，不能让每句解释都显得同样用力。
- 短语优于完整段落。图内文字用于导航与证据，不替代一篇分析文章。

## 交付前的顺序检查

1. 读者在三秒内能否说出图要回答的问题？
2. 图形形式是否与信息关系匹配，而不是只是“看起来像一页PPT”？
3. 是否存在没有语义的色块、箭头、图例或装饰？
4. 是否清楚标明单位、期间、估算边界和数据来源？
5. 并列模块是否逐行对齐，正文是否采用有限且规律的间距？
6. 标题、关键数字、正文、注释是否有明确但不过度的字重层级？
7. 在导出尺寸下，正文和脚注是否仍可读？
8. 图表标题区是否与绘图区分离？层级图的同级节点是否仍有可见间隙？
9. 若使用时间轴，事件间的视觉距离是否与真实时间间隔一致？
10. 若使用小倍图，所有可比面板是否共享同一尺度与网格？
11. 若使用构成迁移图，核心中间层是否另有可精确读取的表达？

若其中任一项不能回答“是”，先修结构，再调颜色和细节。
